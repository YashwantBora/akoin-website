Step-by-Step Upload

Go to github.com and sign up / log in

Create a free account if you don't have one. No credit card needed.

Create a new repository

Click the + icon top right → New repository → Name it akoin-website → Set to Public → Tick "Add README file" → Click Create repository

Upload all 4 files

Click Add file → Upload files → Drag and drop: index.html, advisory.html, academy.html, README.md → Click Commit changes

Enable GitHub Pages

Click Settings tab → Scroll left sidebar to Pages → Under Branch select main → Click Save

Wait 1–2 minutes then refresh

Go back to Settings → Pages. You'll see a green banner with your live URL.

Your site is live! Share the link.

Copy the URL and paste it in your submission email to akoininfo@gmail.com

Pre-Upload Checklist

Renamed homepage.html → index.html

GitHub Pages needs index.html as the entry point. Without this the homepage won't load.

All 3 HTML files are in the same folder

index.html, advisory.html, and academy.html must be together so the nav links work.

README.md is ready to upload

Adds a professional description to your GitHub repository page.

GitHub account created and logged in

Sign up free at github.com. No payment required.

Repository created and set to Public

GitHub Pages only works on Public repositories on the free plan.

Files uploaded and committed

After dragging files in, scroll down and click Commit changes — this is the save button.

GitHub Pages enabled in Settings

Settings → Pages → Branch: main → Save. Then wait 1–2 minutes.

Tested the live URL in browser

Open the live link and click through all 3 pages to confirm everything works.
